# docs-ai

Documentação operacional e técnica para evolução segura do projeto `sorteios_worker`.

## Arquivos
- `plano-melhoria.md`: plano mestre por fases.
- `diagnostico-completo.md`: diagnóstico técnico consolidado.
- `arquitetura-atual.md`: mapeamento da arquitetura existente.
- `arquitetura-alvo.md`: arquitetura recomendada (backend, frontend, banco, infra).
- `backlog-priorizado.md`: backlog executável com prioridades, esforço e risco.
- `roadmap-fases.md`: cronograma macro por fase.
- `riscos-e-mitigacoes.md`: matriz de riscos e plano de mitigação.
- `checklist-fase-0-1.md`: checklist operacional para iniciar execução.
- `metricas-e-baseline.md`: baseline e métricas para acompanhar evolução.
- `estrategia-testes.md`: estratégia de testes e regressão.
- `plano-migracoes-banco.md`: diretrizes para migrações versionadas.

## Ordem de leitura recomendada
1. `plano-melhoria.md`
2. `diagnostico-completo.md`
3. `backlog-priorizado.md`
4. `checklist-fase-0-1.md`
5. `plano-migracoes-banco.md`
